Agro Telemetry System

Previous Next
  • Description
  • Technical
  • Project
The Agro Telemetry System (AGRO-TS) is a comprehensive system for remote lab data monitoring. It was designed to maximize the ability of scientists and researchers to collect, store and analyze relevant parameters of agro-related samples remotely, like temperature and relative humidity.

AGRO-TS consists of a data acquisition+transmission module and a web app for data visualization.

Main Features:

‚Ä¢ 30 days battery life (with daily transmission).
‚Ä¢ Up to 4 sensors (temperature and humidity).
‚Ä¢ 10+ years of hourly measurements memory.
‚Ä¢ Wi-Fi connectivity.
‚Ä¢ On-site web-based configuration via Wi-Fi with any device (no need of network).
‚Ä¢ Supports induced sporadic measurements.
‚Ä¢ Micro-USB Chargeable.
The core of this device is the Espressif ESP8266 inside an ESP-12 module, which not only provides a microcontroller and wide flash memory to store data, but also Wi-Fi connectivity.
Staying in deep-sleep mode (60uA) and waking up every hour to take the sensors samples it manages to make the 150mAh battery last as needed.

A state machine handles the sequences of checking the battery status, taking and saving the measurements into the temporal RTC memory (as program RAM is lost during deep-sleep) and to flash once it gets filled (not to wear the flash so quickly). In addition it controls the network and server connection, as well as the data packets transmission

Finally, the device connects to a defined network and sends the data packets via HTTP protocol to the defined server. These parameters can be set in the configuration mode, which generates an AP from the device where any device (PC, tablet, phone) can be connected to modify them with a web based graphic interface.

Flow Calibration Unit

Previous Next
  • Description
  • Technical
  • Project
The Flow Calibration Unit (FCU) is a low-cost working standard for metrology (gas and water meters). ts small dimensions allows it to be easily transported to different places, from laboratories to remote locations where a meter could be placed.

Taking into account this advantages, the FCU comes in handy to small utilities companies that are bounded to rely on remote laboratories, thus having higher time and budget costs.

Composed of three main elements: a standard meter, an electronic interface and sensors, the FCU permits the user to obtain very useful data from the meter being tested, such as flow, volume, pressure, temperature and relative error.
The interface between the meter and the electronics is a 100-step optical rotary encoder which is read by an Atmega328p MCU.
This interface controls a display and buttons which are used by the laboratory technician to control the device, and communicates with two (or more) sensor modules via optical port (connected with optical fibre) or wirelessly via 2,4GHz RF.
RF communication is an additional feature consisting in modules that are be plugged into the boards, controlled through SPI.

The timekeeping is done by a dedicated RTC (DS3231).

All the tests results data is stored into an I2C EEPROM memory (up to 500 tests) and can be read by a PC software and formatted into '.csv'.

In addition, it can be controlled with the mobile app via bluetooth, which also displays all the live data and the results. The data is stored in the phone/tablet too, and synchronised to a cloud drive.

Battery-free bike lights

Previous Next
  • Description
  • Technical
  • Project
Every year, 15 billion batteries are consumed and thrown away, which is very harmful for the environment, not only because of its toxic components, but also for the metals mining needed to manufacture them.
These lights are powered straight from the bike's wheels rotational energy, where a dynamo (generator) is placed next to one of them in order to achieve that.

While the bike is not moving (waiting for traffic lights or so) the energy is taken from a supercapacitor, which will keep the lights working on a five-minute sustained basis.

The device is composed of a frontal light (blinking white LED + 2 turning lights) and a rear light (blinking red LED + brake LED + turning lights) that the power circuitry and logic share the cabinet with. The frontal light also counts with buttons to toggle the turning lights using one thumb, without the need of releasing the handle.
The energy is obtained from a DC motor placed along the wheel rotation. The voltage generated is filtered and then linearly regulated to 5.1V, which will charge the supercapacitor and also supply the energy used by the logic and the lights.

The logic is handled by the inexpensive and reliable AtTiny13, which oscillates a pin to blink the LEDs. It also senses the buttons to control the turning lights.

The last designed version of this project contemplated the addition of an accelerometer (read via bit-banging I2C) to control the brake lights when braking (negative acceleration in the lengthwise axis).

All the casings were modeled using FreeCAD, and 3D printed in ABS plastic.
Development: May 2020 - June 2020.


This was a personal project for learning purposes, which I also used and installed in my own bike and worked -and still works- really good.

I learned to use supercapacitors, this MCU I hadn't used before, and to design complex casings to fit electronics.

Domotic alarm clock

Previous Next
  • Description
  • Technical
  • Project
This device is a programmable alarm clock that allows to control home appliances such as lights and plugs when particular events happen.
It shows the current time and the programmed alarm time, controlled by buttons and a knob. When the alarm is triggered, it emits a sound and turns on a LED strip to light the bedroom, simulating a natural dawn.

As the LED strip controller is placed at a fair distance from the logic board, a wire is used for its supply and RS-485 communication.

The main board also considers the connection of two wireless modules, which provide internet connectivity and enable the communication with many other devices.
The Atmega328p is the core of this system. It handles interrupts, communications and the human interface (display, buttons and encoder). The timekeeping is assigned to an RTC IC (DS3231) that counts with a backup battery.

The RGB LED driver module is handled by the main board, which implies what color and intensity is needed.
Using PWM and opto-couplers (to isolate the logic from the LED strip external supply) it controls the MOSFET power transistors that drive the strip.

Wireless communications were provided in the board design, having the option to add these features in a future. The main goal of this was to increment the amount of appliances, plugs, lights and sensors to connect, in order to create a more complex and functional domotic system.
This was a personal project I started in 2014 to learn programming microcontrollers (the first version used a Microchip PIC16F877 and 7-segment displays), and because domotic has always caught my eye.

Also I wanted to have the lights in my bedroom turned on smoothly to wake up more naturally instead of rushing out. In addition, I planned to make an infrared interface to automatically turn on the heating moments before the alarm triggers, during winter season.

Soldering station

Previous Next
  • Description
  • Technical
  • Project
Modern SMD technology requires specific equipment to be mounted with high quality solders, given their small dimensions and terminals clearance.

This laboratory tool allows to solder electronic components properly, controlling the temperature of the solder iron tip and a hot air gun.

It is fully controlled using knobs, to set the aforementioned temperatures and the air flow as needed.
The solder iron is powered with 24VAC, and controlled by a TRIAC + opto-diac, which is also controlled by an Atmel Atmega8 microcontroller using PWM (not PPM). The hot air gun heat element is controlled in the same way but powered with 220VAC instead.
The microcontroller senses each heating element (solder iron and heat gun) with their respective sensors; a thermocouple and a PTC thermistor, which are analogically processed and amplified. Then, each signal is measured by its ADC and the temperature value is calculated.

The hot air gun flow is controlled with PWM, which varies its fan speed.
This soldering station was a personal project I made in 2017 when I had no budget to buy one. I only managed to buy the solder iron and the hot air gun as spare parts, and the components needed to make the PCB.

It has allowed me to make excellent quality solders in all my subsequent projects and it still does. Without any doubt, it's one of the most useful projects I have done.
